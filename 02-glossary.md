# gstack 용어집

## `/office-hours`

문제 정의와 제품 사고를 다시 세우는 frontdoor skill이에요.
지금 gstack의 가장 중요한 시작점이에요.

## Native OpenClaw Skills

Claude Code 세션 없이 OpenClaw agent에서 바로 쓰는 methodology skill 묶음이에요.
예: office-hours, ceo-review, investigate, retro.

## team mode

각 개발자가 글로벌 gstack를 공유하고 세션 시작 때 조용히 업데이트받는 운영 모드예요.
`./setup --team`과 `gstack-team-init`가 핵심 표면이에요.

## `/document-release`

방금 ship한 변경에 맞춰 README, ARCHITECTURE, CONTRIBUTING, CLAUDE.md, TODO 같은 문서를 다시 맞추는 skill이에요.
현재 gstack에선 release loop의 일부로 보는 편이 맞아요.

## methodology source

gstack를 단순 실행기가 아니라 planning discipline과 workflow를 공급하는 방법론 층으로 보는 관점이에요.
OpenClaw 연동을 이해할 때 특히 중요해요.
